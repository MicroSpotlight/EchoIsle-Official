# Support

Need help with Echo Isle? This page explains the best ways to get support, report bugs, request features, and contact the maintainers.

## Before Opening A Report

Please check:

- The app is updated to the latest available version.
- Your device is connected to the internet when searching, refreshing RSS feeds, or streaming.
- The podcast RSS feed URL opens in a browser and returns RSS or Atom XML.
- The episode has a valid audio URL if playback or download fails.
- There is enough free device storage for downloads.

## Report A Bug

Open a bug report:

https://github.com/MicroSpotlight/EchoIsle-Official/issues/new?template=bug_report.yml

Helpful details:

- App version.
- iOS version.
- Device model.
- What you expected to happen.
- What actually happened.
- Steps to reproduce.
- Screenshots or screen recordings, if useful.
- A public RSS feed URL or episode URL, if the issue is feed-specific.

Please do not include private information, private feed URLs, passwords, payment information, or other sensitive data in public issues.

## Request A Feature

Open a feature request:

https://github.com/MicroSpotlight/EchoIsle-Official/issues/new?template=feature_request.yml

Good feature requests explain the listening workflow, the problem, and what a successful solution would feel like.

## Ask A Question

Use GitHub Discussions for questions, setup help, ideas, and broader product feedback:

https://github.com/MicroSpotlight/EchoIsle-Official/discussions

## Content Or Rights Request

If you are a podcast publisher and have a concern about content shown in Echo Isle, please include:

- Show title.
- RSS feed URL.
- Episode URL, if applicable.
- Description of the concern.
- A way to verify publisher ownership or authority.

See the [Podcast Content and Rights Policy](./content-rights.md).

## Privacy Request

For privacy questions, read the [Privacy Policy](./privacy.md), then open an issue if you still need help:

https://github.com/MicroSpotlight/EchoIsle-Official/issues/new?template=privacy_request.yml
